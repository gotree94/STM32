## 개요

이 문서는 로봇 청소기 개발을 예시로 하드웨어 제품의 기획부터 양산까지 전체 프로세스를 설명합니다.

**총 개발 기간:** 14~18개월

**주요 단계:**
| Phase | 내용 | 기간 |
|-------|------|------|
| Phase 0 | 기획 및 시장조사 | 12~16주 |
| Phase 1 | 콘셉트 설계 | 8~12주 |
| Phase 2 | 상세 설계 | 16~24주 |
| Phase 3 | 시제품 및 검증 | 12~16주 |
| Phase 4 | 양산 준비 | 8~12주 |

## Phase 0: 기획 및 시장조사 (12 ~ 16주)

### 시장조사 및 경쟁분석 (8주)

**담당:** 기획팀, 마케팅팀

| 항목 | 세부 내용 |
|------|----------|
| 시장 규모 분석 | 글로벌 시장 약 $15B (2024), 연평균 성장률 20%+ |
| 경쟁사 분석 | Roborock, Ecovacs, iRobot, Samsung, LG 제품 벤치마킹 |
| 가격대 조사 | Entry: 20 ~ 40만원 / Mid: 50 ~ 80만원 / Premium: 100 ~ 200만원 |
| 소비자 니즈 | LiDAR 네비게이션, 물걸레 기능, 자동 먼지 비움, 앱 연동 |

**주요 이슈:**
- 중국 브랜드(Roborock, Dreame)의 가격 경쟁력 대응 전략
- 차별화 포인트 발굴 (예: 자동 세척, 장애물 회피 AI)

### 타겟 사양 수립 (4주)

**담당:** 기획팀, 시스템 아키텍트

| 카테고리 | 타겟 스펙 (Mid-range 기준) |
|---------|---------------------------|
| 흡입력 | 4,000~6,000Pa |
| 배터리 | 5,200mAh (180분 연속 운전) |
| 네비게이션 | LiDAR + 카메라 SLAM |
| 물걸레 | 전동 회전식, 자동 리프트 |
| 연결성 | Wi-Fi 2.4/5GHz, BLE 5.0 |
| 소음 | <65dB |
| 목표가 | 출고가 55만원 (원가율 35%) |

## Phase 1: 콘셉트 설계 (8 ~ 12주)

### 시스템 아키텍처 정의 (4주)

**담당:** 시스템 엔지니어, HW/SW 리드

```
┌─────────────────────────────────────────────────────────┐
│                    시스템 블록도                          │
├─────────────────────────────────────────────────────────┤
│  [LiDAR] ──┐                                            │
│  [카메라] ──┼── [메인 MCU] ── [모터 드라이버] ── [휠/브러시]│
│  [ToF센서]─┘     (ARM)                                  │
│      │            │                                     │
│      │      [Wi-Fi/BLE 모듈]                            │
│      │            │                                     │
│  [충돌센서]    [클라우드/앱]                              │
│  [절벽센서]                                              │
│  [먼지통센서]   [배터리 BMS] ── [충전 스테이션]            │
└─────────────────────────────────────────────────────────┘
```

**주요 이슈:**
- SLAM 알고리즘: 자체 개발 vs 라이선스 (Camsense, Slamtec)
- AI 장애물 인식: 엣지 NPU 탑재 여부 (비용 +$15~20)

### 산업 디자인 (6주)

**담당:** ID팀 (Industrial Design)

| 주차 | 활동 |
|-----|------|
| 1-2주 | 콘셉트 스케치 (10~20안) |
| 3주 | 3D 렌더링 (3~5안 선정) |
| 4주 | 목업 제작 및 내부 리뷰 |
| 5주 | FGI(Focus Group Interview) |
| 6주 | 최종안 확정, CMF(Color/Material/Finish) 결정 |

**주요 이슈:**
- 먼지통 용량 vs 본체 높이 (가구 하부 진입성)
- 스테이션 크기 vs 기능 (자동 세척 시 물탱크 필요)

## Phase 2: 상세 설계 (16 ~ 24주)

### 기구 설계 (12주)

**담당:** ME팀 (Mechanical Engineering)

| 주차 | 활동 | 산출물 |
|-----|------|--------|
| 1-4주 | 내부 레이아웃, 모터/센서 배치 | 3D CAD (초안) |
| 5-8주 | 상세 설계, 공차 분석, DFM 검토 | 3D CAD (상세) |
| 9-10주 | 금형 설계 협의, 언더컷 검토 | 금형 RFQ |
| 11-12주 | Mockup/Proto 제작 | 시제품 (3D프린팅) |

**핵심 부품 설계:**
- **흡입 모터 어셈블리:** BLDC 모터 + 팬 + 기밀 구조
- **구동부:** 휠 모듈, 사이드 브러시, 메인 브러시
- **물걸레 모듈:** 회전판, 가압 메커니즘, 리프트 기구
- **LiDAR 터렛:** 회전 메커니즘, 광학창 설계

**주요 이슈:**
- 방수/방진 등급 (IPX4 이상 필요시 구조 복잡도 증가)
- 소음 저감 (진동 절연, 흡음재 배치)

### 하드웨어 설계 (16주)

**담당:** EE팀 (Electrical Engineering)

| 보드 | 주요 구성 | 타임라인 |
|-----|----------|---------|
| **메인보드** | AP (Cortex-A53), DDR3, eMMC, Wi-Fi/BLE | 8주 |
| **센서보드** | LiDAR I/F, ToF드라이버, 범퍼 입력 | 4주 |
| **모터보드** | BLDC 드라이버 x4, 전류 센싱 | 4주 |
| **BMS** | 리튬이온 5S, 충방전 보호, 잔량 계측 | 4주 |
| **충전스테이션** | AC-DC (19V/2A), 도킹 감지, 통신 | 3주 |

**설계 흐름:**

```
회로 설계 → 시뮬레이션 → 아트워크 → DFM검토 → Gerber출력
    (2주)      (1주)       (2주)     (1주)      (0.5주)
                                         ↓
                              PCB제작 → SMT → 디버깅
                               (2주)   (1주)   (2주)
```

**주요 이슈:**
- EMI/EMC 대응 (Wi-Fi 안테나 배치, 모터 노이즈 필터링)
- 열설계 (BLDC 드라이버 발열, AP 발열)
- 인증 요구사항 (KC, FCC, CE, TELEC)

### 소프트웨어 설계 (20주, HW와 병행)

**담당:** SW팀 (Embedded/App/Cloud)

| 레이어 | 구성 요소 | 담당 | 기간 |
|-------|----------|-----|------|
| **Firmware** | RTOS, HAL, 드라이버 | 임베디드팀 | 16주 |
| **Middleware** | SLAM, 경로계획, 장애물회피 | 알고리즘팀 | 20주 |
| **Application** | 청소모드, 스케줄, OTA Client | 임베디드팀 | 12주 |
| **모바일앱** | iOS/Android, 맵뷰, 리모컨 | 앱팀 | 16주 |
| **클라우드/OTA** | 디바이스 관리, OTA 서버, 통계 | 서버팀 | 16주 |

**SLAM 알고리즘 옵션:**

| 옵션 | 비용 | 개발기간 | 비고 |
|-----|------|---------|------|
| 자체개발 | 인건비만 | 12개월+ | 핵심 IP 확보 |
| Slamtec | $3~5/unit | 즉시 | 로열티 방식 |
| Camsense | $2~4/unit | 2개월 | 중국향 유리 |

**주요 이슈:**
- 알고리즘 최적화 (제한된 RAM/CPU에서 실시간 SLAM)
- OTA 안정성 (브릭 방지, 롤백 메커니즘)
- 보안 (펌웨어 암호화, 클라우드 인증)

**OTA 시스템 핵심 기능:**

| 기능 | 설명 | 구현 방법 |
|-----|------|----------|
| **A/B 파티션** | 업데이트 실패 시 이전 버전으로 롤백 | Dual boot partition |
| **Delta 업데이트** | 변경분만 다운로드 (용량 80% 감소) | bsdiff/bspatch |
| **단계적 롤아웃** | 1%→10%→50%→100% 점진 배포 | Feature flag |
| **강제 업데이트** | 보안 취약점 패치 시 필수 적용 | Version policy |
| **업데이트 스케줄** | 청소 중이 아닐 때 업데이트 | Idle detection |

**OTA 보안 요구사항:**

| 항목 | 규격 | 비고 |
|-----|------|------|
| 펌웨어 서명 | RSA-2048 / ECDSA P-256 | 코드 무결성 |
| 전송 암호화 | TLS 1.3 | HTTPS 다운로드 |
| 체크섬 | SHA-256 | 손상 검증 |
| Secure Boot | Hardware Root of Trust | 변조 방지 |
| 롤백 보호 | Anti-rollback counter | 다운그레이드 방지 |

**주요 이슈:**
- 대역폭 최적화 (수만 대 동시 다운로드 시 CDN 활용)
- 업데이트 중 전원 차단 대응 (Resume 지원)

## Gate Review 체크리스트

### ✅ Gate 1: 콘셉트 승인

- [ ] 시장 분석 및 경쟁력 검토 완료
- [ ] 타겟 사양 및 목표가 설정
- [ ] ID 콘셉트 승인
- [ ] 초기 BOM 및 원가 추정
- [ ] 서버/앱 아키텍처 초안 검토

### ✅ Gate 2: 설계 완료

- [ ] 기구/회로/SW 상세 설계 완료
- [ ] DFM/DFA 검토 완료
- [ ] 금형 발주 승인
- [ ] 인증 계획 수립
- [ ] 서버 인프라 구축 완료
- [ ] 앱 UI/UX 디자인 확정
- [ ] API 스펙 문서화 완료

### ✅ Gate 3: EVT 완료

- [ ] 기능 검증 Pass
- [ ] Critical 이슈 해결
- [ ] DVT 빌드 승인
- [ ] 앱-디바이스 연동 검증 완료
- [ ] OTA 업데이트 기능 검증

### ✅ Gate 4: DVT 완료

- [ ] 신뢰성 시험 Pass
- [ ] 인증 취득 (또는 일정 확정)
- [ ] 양산 BOM Freeze
- [ ] PVT 빌드 승인
- [ ] 서버 부하 테스트 Pass
- [ ] 앱 스토어 심사 준비 완료
- [ ] 음성 비서 연동 인증 (Alexa, Google)

### ✅ Gate 5: 양산 승인

- [ ] PVT 수율 목표 달성
- [ ] 전체 인증 완료
- [ ] 양산 라인 준비 완료
- [ ] 초도 물량 출하 승인
- [ ] 앱 스토어 출시 완료 (iOS/Android)
- [ ] 서버 프로덕션 환경 검증
- [ ] 고객 지원 시스템 준비 (FAQ, CS 도구)
