# smart_car 프로젝트 분석 리포트

> 프로젝트: STM32F103RBT6 (NUCLEO-F103RB) 기반 SLAM 매핑 로봇  
> IDE: STM32CubeIDE (GCC)  
> HAL 버전: STM32Cube FW_F1 V1.8.7  
> 분석일: 2026-07-01

---

## 1. 프로젝트 개요

본 프로젝트는 STM32F103RBT6 (Cortex-M3, 64MHz) 기반 4륜 구동 로봇으로, <br>
실내 공간의 SLAM(Simultaneous Localization and Mapping) 데이터를 수집하는 것을 목표로 한다. <br>
적외선 리모컨(IR)으로 시작 신호를 받아 **44걸음(step)** 동안 10cm씩 전진하며 <br>
매 정지 시점마다 초음파 센서 2개(서보 모터로 0~180도 스윕)로 거리를 측정하고, <br>
11걸음마다 90도 회전하여 방을 한 바퀴 도는 방식의 **Grid-based SLAM(격자 기반 매핑)** 을 수행한다. <br>
모든 데이터는 USART2(디버그)와 USART3(관제)로 전송된다.

## 2. 1조(SLAM_sample) vs 2조(smart_car) 공통점/차이점

| 항목 | 1조 (SLAM_sample) | 2조 (smart_car) |
|------|-------------------|------------------|
| MCU | STM32F103RBT6 | STM32F103RBT6 |
| 클럭 | 64MHz (HSI+PLL) | 64MHz (HSI+PLL) |
| LCD | ST7735 (160x120) | ST7735 (160x80) |
| 자이로 | MPU-6050 (SW I2C) | MPU-6050 (SW I2C) |
| 모터 | 4개 DC모터 (PWM 10kHz) | 4개 DC모터 (PWM 977Hz) |
| 엔코더 | PB13 포토 인터럽터 | PB13 + PC3 (양轮) |
| 초음파 | 3x HC-SR04 (EXIT) | 2x HC-SR04 (Polling) |
| 서보모터 | 없음 | PA11 서보 (0~180도 스윕) |
| IR 리모컨 | 없음 | PB1 IR 수신 (NEC 프로토콜) |
| 주행 방식 | **연속 자율주행** (슈퍼루프) | **정지-측정-전진** (순차) |
| 상태 머신 | 4-state 자율 회피 | 선형 순차 (44-step) |
| SLAM 전송 | 10Hz 연속 CSV | Step별 일괄 전송 |
| 센서 종류 | IR + TCRT + 불꽃 + CDS + 충격 | 초음파 + 자이로 + 엔코더 + IR |

## 6. 소프트웨어 모듈 분석

### 6.1 `main.c` (897줄) - 메인 로직

#### 초기화 순서
1. HAL_Init → SystemClock_Config → MX_GPIO_Init
2. MX_USART2_UART_Init → MX_TIM4~1_Init → MX_USART3_UART_Init → MX_SPI1_Init
3. DWT_Init (사이클 카운터)
4. MPU6050_Init → 실패 시 i2c_scan()으로 장치 탐색
5. LCD_Init → LCD_ShowStatus(LCD_STATE_IDLE)
6. **IR 대기 루프**: `robot_started` 플래그가 SET될 때까지 IR 수신 대기 (IR_BTN_START = 0xC2)
7. motor_pwm_start() → HAL_Delay(3000)

#### 메인 실행: 44-Step 순차 시퀀스
```c
for (step = 1; step <= 44; step++) {
    LCD_ShowStatus(LCD_STATE_DRIVE);     // "주행"
    test_distance_measure(step);         // 10cm 전진 + 정지 + UART 출력
    LCD_ShowStatus(LCD_STATE_SCAN);      // "스캔"
    stationary_scan();                   // 서보 0→180° 스윕 + 초음파 측정
    if (step % 11 == 0 && step != 44) {  // 11, 22, 33번째
        LCD_ShowStatus(LCD_STATE_ROTATE); // "회전"
        test_rotate_90();                 // 90도 회전
    }
}
LCD_ShowStatus(LCD_STATE_IDLE);          // "대기"
while(1);  // 종료
```

### 6.2 `lcd.c` + `lcd.h` - ST7735 LCD 드라이버 (+한글 폰트)

**특징**:  
- ST7735S 160x80 패널 (Y_OFFSET = 24로 GRAM 오프셋 보정)
- **한글 16x16 비트맵 폰트 내장** (주/행/스/캔/회/전/대/기) → GLYPH_SCALE=3으로 48x48 확대 출력
- 4개 상태 단어를 보라색(PURPLE)으로 화면 중앙 표시
- `LCD_ShowStatus()`: 동일 상태면 재출력하지 않음 (SPI 트래픽 최적화)

**상태 매핑**:
| 열거형 | 단어 | 의미 |
|--------|------|------|
| LCD_STATE_DRIVE | 주행 | 10cm 전진 중 |
| LCD_STATE_SCAN | 스캔 | 초음파 스윕 중 |
| LCD_STATE_ROTATE | 회전 | 90도 코너 회전 중 |
| LCD_STATE_IDLE | 대기 | IR 대기 or 임무 완료 |

### 6.3 `mpu6050.c` + `mpu6050.h` - MPU-6050 드라이버

**함수**:
- `MPU6050_Init()`: WHO_AM_I(0x75) 확인 → 0x68 or 0x98, Power Mgmt 1 레지스터 Wake (0x00), ±250dps 설정
- `MPU6050_ReadGyroZ_dps()`: 레지스터 0x47에서 Z축 자이로 읽기 → 131.0으로 나눔
- `MPU6050_CalibrateGyroZ(samples)`: N회 평균 오프셋 계산
- `MPU6050_ReadAccel_g()`: 레지스터 0x3B에서 가속도 6바이트 읽기 → 16384.0으로 나눔 (±2g)

### 6.4 `soft_i2c.c` + `soft_i2c.h` - Software I2C Bit-bang

- PC4(SCL), PC5(SDA) Open-Drain, Pull-Up
- DWT Cycle 기반 delay_us 정밀 타이밍
- 표준 I2C: Start/Stop/WriteByte(ACK 체크)/ReadByte(ACK/NACK)
- SoftI2C_Init()에서 GPIO 초기화

### 6.5 `slam.c` + `slam.h` - SLAM 스캔 엔진

**핵심 기능**:
- `measure_echo1()` / `measure_echo2()`: TRIG 발사(10us) → Echo Polling (30ms 타임아웃) → pulse width → cm 환산 (÷58)
- `measure_echo1_median()` / `measure_echo2_median()`: 3회 측정 → **중간값(Median) 필터**로 이상치 제거
- `servo_write_angle(angle)`: 서보 PWM 수동 생성 (500~2500us, 50Hz, 2펄스)
- `servo_write_angle_init(angle)`: 큰 각도 이동 시 10펄스로 안정화
- `stationary_scan()`: **0~180도 2도 간격(91포인트)** 스캔:
  ```
  for angle = 0 to 180 step 2:
      servo_write_angle(angle)
      v1[sweep] = measure_echo1_median()  // 3회 중간값
      v2[sweep] = measure_echo2_median()  // 3회 중간값
      final_d1 = median_of_n(v1, N_SWEEP) // N_SWEEP=1 (현재)
      final_d2 = median_of_n(v2, N_SWEEP)
      USART3 송신: "S,<각도>,<거리1>,<거리2>\r\n"
  ```

### 6.6 `stm32f1xx_it.c` - 인터럽트 핸들러

| 인터럽트 | 핀 | 기능 |
|---------|-----|------|
| EXTI1_IRQHandler | PB1 (IR_RX) | NEC 리모컨 신호 수신 |
| EXTI3_IRQHandler | PC3 (TRACK_R) | 우측 엔코더 펄스 |
| EXTI15_10_IRQHandler | PB13 (TRACK_L) | 좌측 엔코더 펄스 |

### 6.7 `HAL_GPIO_EXTI_Callback()` - 통합 라우터

**엔코더 (PB13, PC3)**:
- DWT Cycle 기반 3ms 디바운스
- TRACK_L(PB13) → `pulse_L++`, TRACK_R(PC3) → `pulse_R++`

**IR 리모컨 (PB1 - NEC Protocol)**:
- Fall Edge 인터럽트로 펄스 간격(gap_us) 측정
- **13.5ms (9ms+4.5ms)**: 리더 프레임 → `ir_bitcount = 0`, `ir_code = 0`
- **1.125ms (bit=0)** or **2.25ms (bit=1)**: 32비트 시프트 적재
- 32비트 완료 → `ir_ready = 1`
- 메인 루프에서 cmd byte (code >> 8) & 0xFF 추출

## 7. 상태 머신 분석

이 프로젝트는 **실시간 상태 머신이 아닌 선형 순차 시퀀스**로 동작한다. 아래와 같은 단계가 반복된다:

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  IR 대기  │────→│ LCD INIT │────→│  44-Step │
│ (IDLE)   │     │ (IDLE)   │     │  루프    │
└──────────┘     └──────────┘     └────┬─────┘
                                        │
                   ┌────────────────────┼────────────────────┐
                   ▼                    ▼                    ▼
            ┌──────────┐        ┌──────────┐        ┌──────────┐
            │ STEP n   │        │ STEP 11  │        │ STEP 44  │
            │ 주행→스캔 │        │ 주행→스캔│        │ 주행→스캔│
            └──────────┘        │ →회전→계속│        │ →IDLE    │
                                └──────────┘        └──────────┘
```

### Step 내부 상세

```
┌─────────────────────────────────────┐
│ Step n                              │
│  1. LCD_ShowStatus(DRIVE) = "주행"   │
│  2. test_distance_measure(n):       │
│      - pulse_L = 0, pulse_R = 0     │
│      - motor_forward(TEST_DUTY)     │
│      - while avg_pulse < TARGET_PULSE│
│      - motor_stop()                 │
│      - 거리 = avg_pulse * CM_PER     │
│        + STOP_OVERSHOOT_CM          │
│      - UART: "STEP,n,dist, ..."     │
│  3. LCD_ShowStatus(SCAN) = "스캔"    │
│  4. stationary_scan():              │
│      - 0~180° 2° 간격 초음파 측정    │
│      - USART3: "S,angle,d1,d2"      │
│  5. if (n % 11 == 0):               │
│      - LCD_ShowStatus(ROTATE)="회전" │
│      - test_rotate_90():            │
│        MPU6050_CalibrateGyroZ(200)  │
│        motor_turn_left()            │
│        while yaw < 90° 적분         │
│        motor_stop() + 400ms 대기    │
│        UART: "ROTATE DONE ..."      │
└─────────────────────────────────────┘
```

### IR 대기 루프
```
LCD_ShowStatus(IDLE) = "대기"
while (!robot_started):
    if ir_ready:
        cmd = (ir_code >> 8) & 0xFF
        if cmd == 0xC2: robot_started = 1
```

## 8. 시간적 흐름 (Timing Diagram)

### 초기화 시퀀스 (~6초)
```
 0ms  HAL_Init, SystemClock_Config
 20ms MX_*_Init (GPIO, UART, TIM, SPI)
 30ms DWT_Init
 40ms MPU6050_Init (I2C, WHO_AM_I, Wake)
 80ms LCD_Init (ST7735: RESET→SLPOUT→DISPON, ~850ms)
 930ms LCD_ShowStatus("대기")
       → IR 리모컨 대기 (무한) ← 사용자가 IR 버튼(0xC2) 누를 때까지
       → IR 수신 시 robot_started = 1
+100ms motor_pwm_start()
+3100ms HAL_Delay(3000) — 출발 준비
```

### Step 1회 (~수 초)
```
 0ms   LCD("주행"), motor_forward(), pulse 카운트 시작
 ~500ms avg_pulse ≥ 5 → motor_stop()
       → 거리 계산 → UART 송신
+550ms LCD("스캔"), stationary_scan() 시작
       servo_write_angle_init(0) → 400ms 대기
       for angle=0..180 step 2 (91포인트):
           servo_write_angle(angle)  → ~2ms
           measure_echo1_median()    → 3회 × ~4ms = ~12ms
           measure_echo2_median()    → 3회 × ~3ms = ~9ms
           UART 송신
       ≈ 91 × 25ms ≈ 2275ms
```

### 11-Step 회전 (~2초)
```
 0ms   LCD("회전")
       MPU6050_CalibrateGyroZ(200) → 200 × 2ms = 400ms
       motor_turn_left() 시작
       while yaw < 90° (적분 루프)
       motor_stop() + 400ms 대기
       UART 송신
≈ 1.5~2초
```

### 전체 임무 소요 시간 (44 Step)
```
Step당: ~500ms(주행) + ~2300ms(스캔) ≈ 2800ms
44 Steps × 2800ms ≈ 123초 (약 2분)
+ 3회 회전 × 2000ms ≈ 6초
+ IR 대기 시간 (가변)
≈ 총 2분 30초 ~ 3분
```

### USART3 (SLAM 관제, 115200bps)
```

S,2,16.0,13.5      ← angle=2°, echo1=16.0cm, echo2=13.5cm
...
S,180,999.0,999.0   ← angle=180°, 측정 실패(-1) → 999.0?
STEP,1,10.30,5,5,0.123,0.045,0.987,0.56  ← 전진 데이터
```

## 11. 잘된 부분 (Strengths)

### 11.1 측정 정밀도를 위한 설계
- **Median 필터**: 3회 연속 측정 후 중간값 선택 → 초음파 노이즈 이상치(outlier) 효과적 제거
- **정지 후 측정**: 전진 후 완전 정지 상태에서 초음파 측정 → 모터 진동 영향 제거
- **오버슈트 보정**: `STOP_OVERSHOOT_CM = 5.3cm`로 관성에 의한 오버슈트를 펄스 계산에 반영
- **DWT Cycle 디바운스**: 엔코더 3ms 디바운스를 사이클 카운터로 정밀 구현

### 11.2 명확한 임무 구조
- **44-Step 선형 시퀀스**로 복잡한 상태 머신 없이 단순하고 이해하기 쉬움
- 11걸음마다 90도 회전으로 Grid Mapping에 최적화 (4면 × 11걸음 = 44)
- **한글 LCD UI**: "주행/스캔/회전/대기"로 사용자에게 직관적인 상태 표시

### 11.3 디버깅 인프라
- USART2로 모든 단계의 상세 로그 출력 (펄스 수, 거리, 자이로 오프셋, 시간)
- USART3는 SLAM 데이터 전용 채널로 분리 → 관제 프로그램 데이터 파싱 용이
- I2C 스캔 함수(i2c_scan()) 내장으로 하드웨어 문제 진단 가능

### 11.4 독립적 모듈화
- 각 기능이 **.c/.h 분리**되어 컴파일 단위 분리 (1조의 header-only 방식보다 개선됨)
- `slam.c`/`mpu6050.c`/`soft_i2c.c`/`lcd.c` 각각 독립적

## 12. 개선이 필요한 부분 (Improvements)

### 12.1 구조적 문제 🔴

#### [Critical] 초음파 Polling 방식의 비효율
- `measure_echo1/2()`가 **Busy-wait Polling**으로 Echo 신호 대기 (최대 30ms)
- 스캔 1포인트당 2개 센서 × 3회 × 2개 = 6번 Polling → 약 180ms 블로킹
- 전체 스캔 91포인트 × 180ms ≈ **16초** 동안 CPU가 사실상 대기
- **→ EXTI 인터럽트 + 타임아웃 타이머 방식으로 전환 권장**

#### [High] PWM 주파수
- TIM Prescaler = 0, Period = 65535 → 64MHz / 65536 ≈ **977Hz**
- 모터 구동에 977Hz는 가청 주파수 대역으로 **소음 발생**
- 1조는 10kHz 사용 → **PWM 주파수 상향 권장** (20kHz 이상이면 비가청)

#### [High] 메모리 사용
- sprintf + HAL_UART_Transmit 조합이 함수마다 반복 (스택 사용 증가)
- 약 3초 HAL_Delay(3000)가 초기화 직후에 위치 → 불필요한 대기

### 12.2 안정성 리스크 🔶

#### [Medium] 초음파 타임아웃 고정
- Polling while 루프 타임아웃이 `30000us = 30ms`로 고정
- HC-SR04 최대 측정 거리는 4m(약 24ms) → 여유는 있지만 환경에 따라 조정 필요
- Polling 대기 중 다른 인터럽트(IR, 엔코더)는 처리되나 전체 시스템 응답성 저하

#### [Medium] 엔코더 캘리브레이션 미흡
- `CM_PER_PULSE = 1.0f` (주석: 실측 보정 필요)
- `STOP_OVERSHOOT_CM = 5.3f`도 실측 기반이지만 코드 상수로 고정
- 바퀴 마모, 배터리 전압 등에 따라 변화 → **동적 캘리브레이션 필요**

### 12.3 기능 개선 🔷

#### [Medium] SLAM 스캔 해상도
- `N_SWEEP = 1`로 설정되어 median_of_n이 사실상 단일 값 반환
- 각도당 1회 측정이므로 Median 필터 효과 없음
- **→ N_SWEEP을 3~5로 증가 권장** (단, 스캔 시간 증가 고려)

#### [Low] 모터 duty 고정
- `TEST_DUTY = 32767` (50%)로 고정
- 속도 제어 없이 ON/OFF 방식 → 배터리 전압에 따라 이동 거리 변동

#### [Low] 자이로 Yaw 리셋
- 회전 시 `MPU6050_CalibrateGyroZ(200)`으로 매번 새 오프셋 계산
- 적분 누적 오차를 리셋하지 않음 → 장시간 주행 시 각도 오차 누적

### 12.4 코드 품질 🔷

#### [Low] 중복 코드
- `measure_echo1()`과 `measure_echo2()`가 거의 동일 (ECHO 핀만 다름)
- **→ 함수 포인터 or 매크로 통합 가능**

#### [Low] 하드코딩된 상수
- 매직 넘버: `0xC2`(IR_BTN_START), `131.0f`(자이로 스케일), `16384.0f`(가속도), `58.0f`(초음파)
- 이미 #define 있으나 통일성 부족

### 12.5 개선 우선순위

| 우선순위 | 항목 | 영향 |
|---------|------|------|
| 🔴 P0 | 초음파 Polling → EXTI 방식 | CPU 효율, 응답성 |
| 🔶 P1 | PWM 주파수 10kHz+ 상향 | 소음 감소 |
| 🔶 P1 | N_SWEEP 증가 (3~5) | 측정 정밀도 |
| 🔷 P2 | CM_PER_PULSE 동적 보정 | 거리 정확도 |
| 🔷 P2 | 모터 속도 제어 (가변 duty) | 주행 일관성 |
| 🔷 P3 | 초음파 측정 함수 리팩토링 | 코드 중복 제거 |

## 13. 1조 대비 2조의 특징 비교

| 측면 | 1조 (SLAM_sample) | 2조 (smart_car) |
|------|-------------------|------------------|
| **코드 구조** | Header-only inline (단순 but 중복) | .c/.h 분리 (모듈화 우수) |
| **주행 방식** | 실시간 자율회피 (슈퍼루프) | 순차 Grid Mapping (시퀀스) |
| **센서 활용** | 6종 센서 (IR+TCRT+불꽃+CDS+충격) | 3종 센서 (IR+초음파+자이로) |
| **초음파** | EXTI 인터럽트 (비동기) | Polling (동기 블로킹) |
| **측정 정밀도** | EMA 필터 (1회) | Median 필터 (3회) |
| **SLAM 방식** | 연속 10Hz CSV 스트리밍 | Step별 0~180° 스윕 일괄 전송 |
| **사용자 인터페이스** | 영문 LCD (160x120) | 한글 LCD (160x80) |
| **디버깅** | printf 의존 | HAL_UART_Transmit 직접 사용 |
| **IR 제어** | Bluetooth (w/a/s/d) | IR 리모컨 (NEC Protocol) |
| **전체 완성도** | 자율주행 + 다중 센서 융합 (高) | Grid Mapping 정밀도 (中) |

**요약**: 1조는 **실시간 자율주행 + 다중 센서 융합**에 초점, 2조는 **정밀 Grid Mapping을 위한 순차 측정**에 특화되어 있다. 2조는 코드 모듈화와 측정 필터링에서 앞서지만, 초음파 Polling 방식으로 인한 CPU 효율 저하가 주요 개선점이다.
