# SLAM_sample 프로젝트 분석 리포트

> 프로젝트: STM32F103RBT6 (NUCLEO-F103RB) 기반 자율주행 SLAM 로봇  
> IDE: STM32CubeIDE (GCC)  
> HAL 버전: STM32Cube FW_F1 V1.8.7  
> 분석일: 2026-07-01

---

## 1. 프로젝트 개요

본 프로젝트는 STM32F103RBT6 (Cortex-M3, 64MHz)를 기반으로 한 4륜 구동 자율주행 로봇의 펌웨어이다. <br>
초음파 센서 3개(HC-SR04), <br>
MPU-6050 자이로, <br>
적외선 장애물 센서, <br>
TCRT5000 바닥 센서(절벽 감지), <br>
불꽃 감지 센서, <br>
CDS 조도 센서, <br>
충격 센서 등을 활용하여 실내 환경을 탐사하며 <br>
SLAM(Simultaneous Localization and Mapping) 데이터를 UART(115200bps)를 통해 <br>
상위 파이썬 관제 프로그램으로 전송한다.<br>

## 3. 주변장치 설정

| IP | Mode | 상세 설정 |
|------|------|-----------|
| **ADC1** | Scan mode, SW trigger | CH0 (온도센서), CH10 (CDS 조도) |
| **SPI1** | Master, 2-Line | 16Mbps, 8bit, CPOL=0, CPHA=0, NSS Soft |
| **TIM1** | PWM | CH2 (LB1-PA9), CH3 (FL1-PA10) |
| **TIM2** | PWM | CH2 (FL2-PB3), CH3 (LB2-PB10) |
| **TIM3** | PWM | CH1 (FR2-PB4), CH2 (FR1-PB5) |
| **TIM4** | PWM | CH3 (RB1-PB8), CH4 (RB2-PB9) |
| **USART2** | Async TX/RX | 115200-8N1 |
| **TIM Prescaler** | 64-1 | 64MHz/64 = 1MHz → Period 100 → 10kHz PWM |
| **NVIC** | Priority Group 4 | EXTI0/4/15_10 우선순위 0 |

## 5. 소프트웨어 모듈 분석

### 5.1 `main.c` - 메인 루프 및 초기화

#### 초기화 순서
1. HAL_Init() → SystemClock_Config() → MX_GPIO_Init() → MX_USART2_UART_Init()
2. MX_TIM3_Init → MX_TIM1_Init → MX_TIM2_Init → MX_TIM4_Init (PWM 타이머)
3. MX_ADC1_Init → MX_SPI1_Init (ADC, LCD SPI)
4. HAL_Delay(2000) — 안정화 대기
5. Sonar_Init() — DWT 사이클 카운터 초기화
6. Motor_Init() — 8채널 PWM 모두 Start
7. LCD_Init() — ST7735 초기화 (SW Reset, Sleep Out, Display On 등)
8. MPU6050_Init() → MPU6050_CalibrateGyro()

#### 메인 루프 (슈퍼루프, ~수백 Hz)
```c
while(1) {
    EnvSensors_Update();      // 환경 센서 폴링
    Sonar_Scheduler();        // 초음파 TRIG 발사 (25ms 주기)
    MPU6050_Update();         // 자이로 각도 적분
    Drive_System_Update();    // 통합 주행 제어 (핵심)
    Feedback_Update();        // 부저/피드벡 PWM 생성
    // 100ms마다 SLAM CSV 데이터 UART 출력
}
```

#### SLAM 데이터 전송 프로토콜
```
SLAM,<거리(cm)>,<요각(deg)>,<우측초음파(cm)>,<좌측초음파(cm)>,<전방초음파(cm)>
예: SLAM,12.34,45.67,30.5,999.0,25.1
```

### 5.2 `motor_drive.h` - 통합 주행 제어 (핵심 상태 머신)

**위치**: 별도 .c 없이 header-only inline 함수  
**핵심 함수**: `Drive_System_Update(UART_HandleTypeDef *huart)`

#### 주요 기능
- UART ORE(Overrun Error) 클리어
- 3개 초음파 거리값 노이즈 필터링 (0.1cm 미만 → 999.0f)
- 전방/좌/우 장애물 경고 판단
- 전조등 자동 제어 (Headlight_Update)
- **불꽃 감지 시 2초 정지 시퀀스** (Laser ON, Buzzer, 온도 ADC 측정, UART 송신)
- **절벽 감지 시 긴급 후진 + 부저**
- **충돌(충격or4cm이내) 감지 시 긴급 후진 + 부저**
- **블루투스 명령 수신** (타임아웃 1ms)
- **자율/수동 모드 전환**

### 5.3 `MPU-6050.h` - Software I2C 자이로 드라이버

**위치**: Header-only, Software I2C Bit-bang  
**특징**:
- I2C 핀 Open-Drain 직접 제어 (PC6=SDA, PC8=SCL)
- I2C 락업 방지 9-Clock Flush (InitPins)
- 500회 샘플 Gyro Z축 Bias 캘리브레이션
- Yaw 각도 적분 (속도 * dt 누적)
- 노이즈 필터: 1dps 미만 무시
- G-Force 충돌 감지 (5G 이상 → emergency_shock_flag)
- **통신 실패 시 Auto-Recovery** (1초마다 MPU6050_Recover 시도)

### 5.4 `HC-SR04.h` - 초음파 센서 드라이버

**위치**: Header-only  
**3개 센서**: 우측(0번, PB0), 좌측(1번, PA4), 정면(2번, PC11)  
**동작 방식**:
- DWT Cycle Counter로 us 단위 시간 측정 (SystemCoreClock 기준)
- 25ms 간격으로 1개씩 순차 TRIG 발사 (라운드로빈)
- Echo EXTI Rising/Falling Edge로 pulse width 측정
- **안전장치**:
  - 150us~25000us 범위 밖 노이즈 차단
  - EMA 필터 (이전 40% + 현재 60%)
  - 100ms 타임아웃 → 999.0f 반환
  - 초기값(999.0f)이면 즉시 새로운 값 반영

### 5.5 `Encoder.h` - 포토 인터럽터 엔코더

**위치**: Header-only  
**동작**:
- PB13 EXTI Rising Edge 인터럽트
- 5ms 디바운스 필터
- 방향 반영: 전진(dir=1) → ticks++, 후진(dir=-1) → ticks--
- 회전(dir=0) → 카운트하지 않음
- 거리 = ticks * 2.1598cm (5.5cm 바퀴, 8슬릿 기준)

### 5.6 `env_sensors.h` - 환경 센서 통합

**위치**: Header-only  
**센서**:
- IR 사각지대 좌/우 (Active-Low: RESET=장애물)
- TCRT5000 절벽 좌/우 (Active-High: SET=절벽)
- 불꽃 감지 (3회 연속 5ms 간격 디바운스, Active-High)
- CDS 조도 (ADC CH10, 200ms 주기, <3000 ⇒ 어두움)

### 5.7 `feedback_devices.h` - 피드백 장치

**위치**: Header-only, DWT Cycle 기반 수동 PWM  
**기능**:
- Buzzer_EmergencyAlert(): 200ms 동안 2kHz (16000 cycle 토글)
- 후진음: 400ms 주기 (200ms ON / 200ms OFF)
- Laser_Targeting(): 레이저 ON/OFF
- Headlight_Update(): 어두우면 RGB LED ALL ON (White)

### 5.8 `LCD_SPI.h` - ST7735 LCD 드라이버

**위치**: Header-only, SPI 통신  
**해상도**: 160x120, 16bit color (RGB565)  
**제공 함수**: LCD_Init, LCD_Fill, LCD_DrawPixel, LCD_DrawChar, LCD_DrawString  
**폰트**: 8x8 비트맵 폰트 내장 (ASCII 32~126)  
**글자 크기**: 2배 확대 (16x16px)

### 5.9 `display_ui.h` - UI 상태 표시

- UI_ShowStatus(): LCD에 두 줄 텍스트 표시, 내용 변경 시에만 갱신 (깜빡임 방지)
- UI_UpdateStatusText(): 모드명+속도 문자열 포매팅

### 5.10 `shock_sensor.h` - 충격 센서

- PB12 Falling Edge 인터럽트
- 1500ms 쿨다운 디바운스
- emergency_shock_flag SET

## 6. 상태 머신 상세 분석

### 6.1 자율 주행 상태 머신 (`motor_drive.h`)

```
                    ┌─────────────┐
     장애물(<25cm)  │             │  회전 완료
   ┌──────────────→ │  State 0    │ ←───────────┐
   │                │  (직진)     │              │
   │                │             │              │
   │                └──────┬──────┘              │
   │                       │                     │
   │                       │ stop()              │
   │                       ▼                     │
   │                ┌─────────────┐              │
   │                │  State 3    │              │
   └────────────────│  (Scan/판단) │              │
                    │  600ms 대기  │              │
                    └──────┬──────┘              │
                           │                     │
              ┌────────────┼────────────┐        │
              ▼            ▼            ▼        │
       ┌──────────┐ ┌──────────┐  trap?   │      │
       │ State 1  │ │ State 2  │  반대방향│      │
       │ (좌회전) │ │ (우회전) │  강제변경│      │
       │ 75° or   │ │ 75° or   │          │      │
       │ 165°(trap)│ │ 165°(trap)│         │      │
       └────┬─────┘ └────┬─────┘          │      │
            │             │               │      │
            └──────┬──────┘               │      │
                   │ 회전 완료             │      │
                   └──────────────────────┴──────┘
```

**State 0 - 직진 (Forwards)**
- 전방 초음파 25cm 이내 장애물 감지 시 State 3으로 전이
- 정상 시 car_speed 속도로 직진, Encoder 방향 = 1

**State 3 - 스캔/판단 (Stop + 600ms)**
- 600ms 정지 후 좌/우 초음파 거리 측정
- **덫(Trap) 감지**: 마지막 회전 후 2초 이내 → 반대 방향 165° 회전
- **연속 동일 방향 감지**: 3회 연속 같은 방향 → 강제 반대 방향
- 좌/우 거리 비교하여 더 넓은 방향으로 선회 결정
- 좌/우 센서 고장(>200cm) 시 반대쪽 값으로 대체

**State 1 - 좌회전 (Left Spin Turn)**
- 목표 각도: 일반 75°, 덫 감지 시 165°
- 상대 각도 = current_yaw - start_yaw
- 3초 타임아웃 안전장치
- 회전 완료 시 State 0 복귀

**State 2 - 우회전 (Right Spin Turn)**
- 목표 각도: 일반 75°, 덫 감지 시 165°
- 상대 각도 = start_yaw - current_yaw
- 3초 타임아웃 안전장치

### 6.2 불꽃 감지 시퀀스

```
flame_seq_active = 0
    ↓ (env.target_found = 1)
flame_seq_active = 1, stop(), Laser ON, Buzzer
    ↓ ADC 온도 측정 → printf("FIRE_ALERT,TEMP:...")
    ↓ 2초 대기 (주행 차단)
flame_seq_active = 0, Laser OFF → 자율주행 재개
```

### 6.3 충돌/충격 긴급 대응

```
emergency_shock_flag = 1 OR 전방 ≤ 4cm
    ↓
stop() → printf("CRASH") → Buzzer
    ↓ 후진(60%) 700ms
    ↓
auto_mode == 1 → State 3 (스캔)
auto_mode == 0 → car_state = 'x' (정지)
```

### 6.4 절벽 감지 대응

```
cliff_l OR cliff_r
    ↓
stop() → Buzzer → 후진(60%) 500ms
    ↓
auto_mode == 1 → State 3
auto_mode == 0 → car_state = 'x'
```

## 7. 시간적 흐름 (Timing Diagram)

### 초기화 시퀀스 (~5초)
```
0ms    HAL_Init, SystemClock_Config
20ms   MX_*_Init (GPIO, UART, TIM, ADC, SPI)
2020ms HAL_Delay(2000) - 안정화
2030ms Sonar_Init (DWT)
2035ms Motor_Init (PWM Start)
2040ms LCD_Init (ST7735): RESET 100ms → SWRESET 150ms → SLPOUT 500ms → DISPON 100ms
2600ms "SYSTEM BOOTING" 표시
2610ms I2C_InitPins (9-Clock Flush)
2620ms MPU6050_Init (Sleep 해제)
2670ms "CALIBRATION" → MPU6050_CalibrateGyro (~500샘플 × 3ms = 1.5초)
4200ms "SYSTEM READY" → HAL_Delay(500)
4700ms 메인 루프 진입
```

### 메인 루프 1사이클 (약 1~10ms, 가변)
```
Step 1: EnvSensors_Update (<1ms)
        - IR, TCRT, 불꽃 즉시 read
        - CDS는 200ms 주기로만 ADC read
Step 2: Sonar_Scheduler (<1ms)
        - 25ms마다 1개 TRIG 발사 → EXTI로 Echo 대기
Step 3: MPU6050_Update (~2ms I2C read)
        - 14바이트 I2C read, Yaw 적분
Step 4: Drive_System_Update (1~5ms)
        - UART 수신(타임아웃 1ms)
        - 상태 머신 판단
        - PWM duty 갱신
Step 5: Feedback_Update (<0.1ms)
        - DWT cycle 기반 부저 토글
Step 6: SLAM 데이터 출력 (100ms 주기)
        - printf("SLAM,%.2f,%.2f,%.1f,%.1f,%.1f\r\n", ...)
```

### 초음파 타이밍
```
T=0ms    Sensor 0 TRIG → Echo 대기 (최대 24ms=4m)
T=25ms   Sensor 1 TRIG
T=50ms   Sensor 2 TRIG
T=75ms   Sensor 0 TRIG (25ms × 3sensors = 75ms 사이클)
```
각 Echo Rising/Falling Edge는 EXTI 인터럽트로 즉시 처리.

## 9. 잘된 부분 (Strengths)

### 9.1 안정성 중심 설계
- **3중 충돌 방지**: 충격 센서(하드웨어) + MPU-6050 G-Force(가속도) + 초음파 4cm(거리)
- **초음파 노이즈 필터**: 물리적 범위 제한(150us~25000us) + EMA 필터 + 타임아웃(100ms) → 999.0f 반환
- **I2C 복구**: 9-Clock Flush + MPU6050_Recover로 통신 락업 회복
- **UART ORE 클리어**: 오버런 에러 자동 복구
- **무한 회전 방지**: 3초 타임아웃 + 상대각 기준 회전 판단

### 9.2 모듈화
- 모든 드라이버가 **Header-only inline**으로 단일 파일 의존성, include만으로 사용 가능
- 메인 루프가 5개 함수 호출로 단순하고 명확

### 9.3 효율적인 PWM 주파수
- 10kHz PWM으로 모터 구동 소음 최소화
- DWT Cycle Counter 활용으로 추가 타이머 없이 us 정밀도 구현

### 9.4 SLAM 데이터 인터페이스
- 10Hz 간격으로 CSV 형식 데이터 UART 전송 → 파이썬 관제 프로그램과 쉬운 연동
- `printf` 재정의로 UART 출력 자동 라우팅

### 9.5 불꽃 감지 시퀀스
- 2초 정밀 계측: 온도 ADC + 레이저 + 부저 + UART 알림 → 주행 완전 제동
- 관제 통신 1회만 전송하여 통신 폭주 방지

## 10. 개선이 필요한 부분 (Improvements)

### 10.1 구조적 문제 🔴

#### [Critical] Header-only 설계의 한계
- 모든 코드가 `.h` 파일의 `static inline` 함수로 구현되어 있어 **코드 중복 증가** (include마다 별도 인스턴스)
- 상태 변수(`static`)가 각 include 단위마다 별도 복사 → 디버깅 및 유지보수 어려움
- **→ 별도 `.c/.h` 분리 권장**

#### [Critical] 슈퍼루프 실시간성 부재
- 단일 while(1) 루프로 모든 작업 순차 처리
- I2C 통신 중 블로킹 시 전체 시스템 지연
- **→ Timer Interrupt or RTOS 도입 검토**

### 10.2 안정성 리스크 🔶

#### [High] HAL_Delay() 사용
- 메인 루프 내 `HAL_Delay()` (env_sensors 5ms×3, ADC 등) 사용으로 **실시간성 저하**
- `HAL_Delay()`는 SysTick 인터럽트에 의존하므로 인터럽트 내에서 호출 시 교착 위험
- **→ 비블로킹 타이머로 대체 권장**

#### [Medium] 매직 넘버 상수화 부족
- `25.0f`(장애물 기준), `16000`(부저 주기), `50`(I2C 딜레이) 등 상수 정의 없음
- **→ #define 또는 const 변수로 정의**

#### [Medium] 타이머 설정 통일성
- TIM1,2,4: AutoReloadPreload = DISABLE, TIM3: ENABLE → 일관성 부족
- 동일한 Prescaler(64-1), Period(100-1) → 통합 구조체 또는 공통 define 가능

### 10.3 기능 개선 🔷

#### [Medium] 엔코더 정밀도
- 5.5cm 바퀴, 8슬릿 = 1회전당 8펄스 → 약 2.16cm/pulse 분해능
- SLAM 정밀도 향상을 위해 **고해상도 엔코더 or 모터 엔코더 활용 권장**

#### [Low] LCD 성능
- LCD_Fill()이 모든 픽셀을 개별 SPI 전송 (160x120 = 19,200회)
- **→ DMA + SPI 연속 전송으로 개선 가능**

#### [Low] ADC 활용
- ADC CH0 (온도)는 불꽃 감지 시에만 사용, CH10 (CDS)은 200ms 주기
- **→ DMA + Continuous Conversion 모드로 CPU 부하 경감 가능**

### 10.4 코드 품질 🔷

#### [Low] printf 사용
- `printf`는 메모리와 스택 사용량이 큼 (newlib heap 필요)
- **→ sprintf + HAL_UART_Transmit 직접 사용으로 경량화 가능**

#### [Low] 주석
- 코드에 한글 + 이모지 주석이 혼재되어 있음 → 유니코드 인코딩 이슈 가능성
- 일부 주석은 추측성 내용 포함

### 10.5 개선 우선순위

| 우선순위 | 항목 | 영향 |
|---------|------|------|
| 🔴 P0 | .c/.h 분리 리팩토링 | 유지보수성, 코드 크기 |
| 🔴 P0 | HAL_Delay 제거 | 실시간성 |
| 🔶 P1 | 매직 넘버 상수화 | 가독성, 유지보수 |
| 🔶 P1 | 타이머 설정 통일 | 일관성 |
| 🔷 P2 | 엔코더 분해능 향상 | SLAM 정밀도 |
| 🔷 P3 | ADC DMA 도입 | CPU 부하 경감 |

## 11. 종합 평가

본 프로젝트는 STM32F103의 자원 제한(128KB Flash, 20KB RAM)을 고려하여 **header-only 설계로 극단적인 모듈화**를 달성했다. 특히 충돌 방지, I2C 복구, 초음파 노이즈 필터링 등 **안정성 측면에서 상당히 체계적인 방어 로직**이 구현되어 있다. SLAM 데이터 연동 및 자율 주행 상태 머신의 Trap 감지, 연속 동일방향 회전 방지 등 지능적인 판단 로직도 인상적이다.

다만 **header-only 구조로 인한 코드 중복과 디버깅 어려움**, **슈퍼루프의 실시간성 한계**, **HAL_Delay 남용** 등의 구조적 개선점이 존재한다. 규모가 커지거나 기능이 추가될 경우 RTOS 도입과 .c/.h 분할 리팩토링이 필수적으로 검토되어야 한다.
