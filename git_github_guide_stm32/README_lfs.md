# Git LFS (Git Large File Storage) 정리

## 📌 Git LFS란 무엇인가?
Git LFS는 **Git에서 큰 파일을 효율적으로 관리하기 위한 확장 기능**입니다.

### ❗ 기본 Git의 문제
- Git은 모든 파일의 **전체 히스토리**를 저장함  
- 이미지, 영상, 모델 파일 같은 **대용량 바이너리 파일**은 수정할 때마다 통째로 저장됨  
- 저장소 크기 증가 및 clone/pull 속도 저하 발생  

### ✅ Git LFS의 해결 방식
- 실제 큰 파일은 Git 저장소 외부(LFS 서버)에 저장  
- Git에는 **작은 포인터 파일**만 저장  

👉 결과:
- clone 속도 향상  
- 저장소 용량 감소  
- 필요한 시점에만 파일 다운로드  

---

## 🧠 동작 방식
1. `git add` 시 → 포인터 파일이 Git에 저장됨  
2. 실제 파일 → LFS 서버에 업로드됨  
3. `git checkout` 시 → 필요한 파일만 다운로드  

👉 “코드 + 링크 구조”로 동작

---

## ⚙️ 기본 사용 방법

### 1️⃣ 설치
```bash
git lfs install
```

### 2️⃣ 추적할 파일 지정
```bash
git lfs track "*.psd"
```

### 3️⃣ 커밋 & 푸시
```bash
git add .gitattributes
git add file.psd
git commit -m "Add file"
git push
```

---

## 📦 사용이 적합한 경우
- 🎮 게임 개발 (Unity, Unreal 에셋)
- 🎬 영상 / 이미지 프로젝트
- 🤖 머신러닝 데이터셋 및 모델
- 🎨 디자인 파일 (PSD, AI 등)

👉 **텍스트가 아닌 대용량 파일이 많을 때 적합**

---

## 👍 장점 vs ⚠️ 단점

### 👍 장점
- 저장소 경량화
- clone / pull 속도 향상
- 기존 Git workflow 유지 가능

### ⚠️ 단점
- 별도 설치 필요
- 저장 용량 제한 존재
- 팀원 전체 설정 필요

# 📦 Git LFS 용량 및 요금 (GitHub 기준)

## 1️⃣ 기본 무료 용량
- 저장 용량: **10GB**
- 다운로드 트래픽: **10GB / 월**

👉 저장 + 다운로드 각각 별도 계산

---

## 2️⃣ 무료 초과 시 동작

### ❌ 결제 설정 없음
- LFS 업로드 차단
- 실제 파일 대신 포인터만 다운로드됨

### ✅ 결제 설정 있음
- 계속 사용 가능
- 초과분만 과금

---

## 3️⃣ 추가 비용 (Pay-as-you-go)
- 저장: **$0.07 / GB / 월**
- 다운로드: **$0.0875 / GB**

👉 사용한 만큼만 비용 발생

---

## 4️⃣ 플랜별 기본 제공량

| 플랜 | 기본 용량 |
|------|----------|
| Free / Pro | 10GB |
| Team / Enterprise | 250GB |

---

## ⚠️ 중요한 주의사항

### 🔥 1. 파일 수정 시 용량 누적
- 1GB 파일 수정 → 추가로 1GB 저장됨  
👉 버전마다 용량 계속 증가  

### 🔥 2. 다운로드도 과금
- 팀원의 pull  
- CI/CD (GitHub Actions 등)  
👉 모두 bandwidth 사용량에 포함  

---

## 🧠 용량 확장 방법

✔ 확장 가능  

방법:
1. GitHub 결제수단 등록
2. 초과 사용량 자동 과금

👉 별도 업그레이드 과정 없음

## 🎯 한 줄 요약
**Git LFS는 대용량 파일을 외부 저장소에 두고 Git에는 포인터만 저장하여 성능과 효율을 개선하는 도구이며, 기본 10GB 무료 이후에는 사용량 기반으로 확장 가능하다.**
